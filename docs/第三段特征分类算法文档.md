# 第三段物理意义分类算法文档（V4）

> 本文对应代码：`rail_binning_algorithm_v4.py` → `RailBinningCoreV4._calculate_physical_classification_feature()`
> 注意：基础版 `rail_binning_algorithm.py` 的段3 用的是「直线度拟合」，**只有 V4 版**的段3 才用本文的物理4类分类。

## 算法概述

DZ 四段算法中，**段3（P9–P16）** 是产品形变最大、点位分布最广的一段，单一阈值难以精确刻画。V4 版不再用直线度拟合，而是基于**物理形态特征**将段3 分为 4 类：

| 类别 | 含义 | 几何形态 |
|------|------|---------|
| `FLAT` | 平缓型 | 标准差小，几乎无变形 |
| `ARC_UP` | 上凸圆弧 | 整体上升趋势 |
| `ARC_DOWN` | 下凹圆弧 | 整体下降趋势 |
| `WAVE` | 剧烈波动 | 前后趋势剧变、离散程度大 |

## 数据范围

- **段3 配置点**：P9–P16（8 个点），用于计算 `std_dev`
- **趋势/斜率计算用到 P17**：`trend`、`slope_right` 均用到 P17（P17 属于段4 的起点，但参与段3 的形态判断）

## 特征值计算

对每行数据计算 3 组物理特征（5 个标量）：

```python
p_values = row[['P9','P10','P11','P12','P13','P14','P15','P16']].values  # 段3 八个点

# 特征1：整体趋势斜率（P9→P17）
trend = (row['P17'] - row['P9']) / 8

# 特征2：段内标准差（P9-P16 的离散程度）
std_dev = np.std(p_values)

# 特征3：前后半段斜率差（识别剧烈转折，WAVE 专用）
slope_left  = (row['P13'] - row['P9'])  / 4   # 前半段 P9→P13 斜率
slope_right = (row['P17'] - row['P13']) / 4   # 后半段 P13→P17 斜率
slope_diff  = abs(slope_right - slope_left)
```

## 分类逻辑（优先级顺序，逐条短路）

```python
if std_dev < 0.03:                          # 规则1：波动极小 → 平缓
    category = 'FLAT'
elif trend > 0.015:                         # 规则2：上升趋势明确 → 上凸
    category = 'ARC_UP'
elif trend < -0.015:                        # 规则3：下降趋势明确 → 下凹
    category = 'ARC_DOWN'
elif std_dev >= 0.08:                       # 规则4：剧烈波动（无明确趋势）→ 波动
    category = 'WAVE'
else:                                       # 规则5：中等波动+弱趋势的兜底
    if slope_diff > 0.10:
        category = 'WAVE'                   # 转折陡峭 → 波动
    else:
        category = 'ARC_UP' if trend >= 0 else 'ARC_DOWN'  # 否则按趋势方向归圆弧
```

**设计要点**：
- `FLAT` 优先级最高——只要波动足够小，无论趋势如何都判为平缓。
- `ARC_UP/DOWN` 优先于 `WAVE`：趋势明确时按方向归类，避免把单调曲线误判为波动。
- 兜底分支用 `slope_diff` 区分「中等波动但有陡峭转折（WAVE）」与「中等波动且方向一致（按 trend 归 ARC）」。

## 阈值参数

| 参数 | 值 | 用途 |
|------|----|------|
| `std_dev` 平缓上限 | `0.03` | `std_dev < 0.03` → FLAT |
| `trend` 上升阈值 | `0.015` | `trend > 0.015` → ARC_UP |
| `trend` 下降阈值 | `-0.015` | `trend < -0.015` → ARC_DOWN |
| `std_dev` 波动下限 | `0.08` | `std_dev >= 0.08` → WAVE |
| `slope_diff` 转折阈值 | `0.10` | 兜底分支中 `slope_diff > 0.10` → WAVE |

> 这些阈值硬编码在 `_calculate_physical_classification_feature()` 中。调整时直接修改该方法内的比较值。

## 输出结果

分类完成后写入 `self.df_processed`：

```python
df['seg3_category']       # 分类标签：FLAT / ARC_UP / ARC_DOWN / WAVE / UNKNOWN
df['seg3_trend']          # 整体趋势斜率
df['seg3_std_dev']        # 段内标准差
df['seg3_slope_left']     # 前半段斜率
df['seg3_slope_right']    # 后半段斜率
df['seg3_slope_diff']     # 斜率差
```

## 异常处理

当某行 P9–P16 存在缺失值，或计算过程抛异常时，该行 `seg3_category = 'UNKNOWN'`，各特征值置为 `np.nan`：

```python
if pd.isna(p_values).any():
    classifications.append('UNKNOWN')
    feature_details.append({'trend': np.nan, 'std_dev': np.nan, ...})
    continue
```

## 与基础版的区别

| 项 | 基础版 `rail_binning_algorithm.py` | V4 版 `rail_binning_algorithm_v4.py` |
|----|------------------------------------|--------------------------------------|
| 段3 方法 | `straightness_fit`（直线度拟合，取最大偏差，保留符号） | `physical_classification`（物理4类） |
| 段3 输出 | 单个特征值 `e3` + 标签 `label3`（P/N） | `seg3_category`（4类）+ 5个物理特征 |
| 标签粒度 | 二值 P/N | 四值 FLAT/ARC_UP/ARC_DOWN/WAVE |
