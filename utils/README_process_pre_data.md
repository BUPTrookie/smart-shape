# 预处理数据处理工具使用说明

## 功能概述

`process_pre_data.py` 工具用于处理Data目录下的total.csv文件，将状态为"pre"的数据从h1字段转移到h0字段，并清空h1字段。

## 主要功能

1. **数据转移**: 将RS1h1, RS2h1, RS3h1, RS4h1的数据转移到对应的RS1h0, RS2h0, RS3h0, RS4h0
2. **数据清空**: 转移后清空h1字段的数据（设置为None）
3. **自动备份**: 处理前自动备份原始文件
4. **数据验证**: 处理后进行数据完整性检查
5. **详细日志**: 记录处理过程和统计信息

## 使用方法

### 1. 基本使用（默认参数）

```bash
cd utils
python process_pre_data.py
```

默认处理 `Data/total.csv` 文件，并覆盖原文件，同时创建备份。

### 2. 指定输入文件

```bash
python process_pre_data.py Data/total.csv
```

### 3. 指定输入和输出文件

```bash
python process_pre_data.py Data/total.csv Data/total_processed.csv
```

处理后的数据保存到新文件，原文件保持不变。

### 4. 禁用备份（不推荐）

```bash
python process_pre_data.py Data/total.csv Data/total_processed.csv false
```

## 命令行参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 第1个参数 | 输入文件路径 | `Data/total.csv` |
| 第2个参数 | 输出文件路径（可选） | 覆盖原文件 |
| 第3个参数 | 是否创建备份（true/false） | `true` |

## 数据处理逻辑

### 处理条件
- 只处理Status列值为 "Pre" 的行数据
- 处理字段：RS1h1, RS2h1, RS3h1, RS4h1 → RS1h0, RS2h0, RS3h0, RS4h0

### 处理步骤
1. **数据验证**: 检查必要的列是否存在
2. **备份创建**: 自动创建带时间戳的备份文件
3. **数据转移**: h1 → h0
4. **数据清空**: 将h1字段设置为None
5. **完整性验证**: 检查处理结果的正确性
6. **结果保存**: 保存处理后的数据

### 备份文件命名规则
```
原文件: Data/total.csv
备份文件: Data/total_backup_20231214_143052.csv
```

## 错误处理

### 常见错误及解决方案

1. **文件不存在错误**
   ```
   FileNotFoundError: 输入文件不存在: Data/total.csv
   ```
   **解决**: 确保Data目录下存在total.csv文件

2. **缺少必要列**
   ```
   ValueError: 缺少必要的列: ['Status', 'RS1h1', ...]
   ```
   **解决**: 检查CSV文件是否包含所有必要的列

3. **权限错误**
   ```
   PermissionError: 无法创建备份文件
   ```
   **解决**: 检查目录写入权限

## 注意事项

1. **数据安全**: 工具默认会创建备份，建议在大数据处理前先测试
2. **内存使用**: 大文件处理时会占用相应内存
3. **编码支持**: 自动检测UTF-8和GBK编码
4. **日志记录**: 详细记录处理过程，便于问题排查

## 验证方法

处理完成后，可以通过以下方式验证：

1. **检查统计信息**: 工具输出中会显示处理前后的数据统计
2. **查看备份文件**: 对比原始备份和处理后的文件
3. **数据抽样检查**: 手动检查几行pre状态数据的处理结果

## 恢复数据

如果需要恢复到处理前的状态：

```bash
# 查找最新的备份文件
ls Data/total_backup_*.csv

# 恢复数据（替换文件名）
cp Data/total_backup_20231214_143052.csv Data/total.csv
```
